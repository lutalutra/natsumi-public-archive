# Natsumi Sugiyama — Archive of Practice (リファクタリング版)

## これは何か

元の `index_header_about_only_updated (1).html`（1ファイル・6,120行、
`<style>` 15個、`<script>` 38個がすべてインライン）を、**見た目も動作も
一切変えずに**、3つのファイルへ整理し直したものです。

```
index.html   構造（HTML）のみ。<style>と<script>を全て外部ファイル参照に変更。
styles.css   元の15個の<style>ブロックを、元の順番のまま1つに結合。
app.js       元の38個の<script>ブロックを、元の順番のまま1つに結合。
```

## 何を変えていないか（検証済み）

- **CSSの中身**：セレクタ、プロパティ、値、記述順序は1文字も変更していません。
  15個のスタイルブロックを元の出現順のまま連結しただけなので、カスケード
  （後勝ちのルール）は完全に同じです。
- **JSの中身**：ロジック、関数、変数名、実行順序は1文字も変更していません。
  38個のスクリプトブロックを元の出現順のまま連結しただけです。ブラウザ上では
  複数の`<script>`タグも1つの`<script>`タグも同じグローバルスコープを共有する
  ため、動作は完全に同一です。
- **HTMLの中身**：`<style>`と`<script>`タグを取り除いた残りの要素・属性・
  テキスト・コメントは元のまま、1文字も変更していません。

上記はいずれも、空白文字を除いた完全一致を機械的に diff して確認済みです
（`app.js` は 287,054 文字、`styles.css` は 58,693 文字、いずれも元ファイル
と完全一致）。

## 何を変えたか

- 1つの巨大な `.html` ファイルに埋め込まれていた CSS / JS を、それぞれ
  `styles.css` / `app.js` という外部ファイルに切り出し、`<link>` と
  `<script src="">` で読み込むようにしました。
- 各ブロックの先頭に「何番目のブロックか／元のid」を示すバナーコメントを
  追加しました（`v15-fixes` のような元の `id` 属性をそのままラベルとして
  使用）。ファイル冒頭には全ブロックの目次コメントを追加しています。
- CSS の読み込み位置を、元は body 内に分散していたものを `<head>` 内の
  1つの `<link>` にまとめました。CSSは記述順にのみ依存し、head/body の
  どちらに置かれるかでは挙動が変わらないため、見た目への影響はありません
  （むしろちらつき防止の観点ではわずかに有利です）。

## あえてやらなかったこと（重要）

このファイル群には、`v15-fixes` から `v70-...` まで、同じ機能を何度も
上書き・修正した形跡のパッチが大量に積み重なっています（例:
`v24-force-list-links` → `v25-real-list-link-fix` →
`v26-restore-project-links` → `v27-final-link-and-drawer-fix` のように、
似た問題を繰り返し直そうとした跡が見えます)。

こうした「実質的に無効化されている古いコード」を削除して本当の意味で
軽量化するには、各パッチが今のDOM/CSSに対して何を上書きしているかを
1つずつ実行時の挙動で確認する必要があり、ブラウザでの目視・操作確認なしに
安全に断定することはできません。今回は「見た目と機能を一切変えない」という
最優先の指示を守るため、その工程（死んでいるコードの安全な削除）は行わず、
**まずは同一の挙動を保証できる整理（ファイル分割・順序保持・ラベル付け）
だけ**を行いました。

次のステップとして、実際にブラウザで動作確認をしながら:

1. 各 `vNN` ブロックが今も効果を持っているか、完全に後続ブロックに
   上書きされて無効化されているかを1つずつ確認し、
2. 無効化されていることが確認できたブロックのみを安全に削除・統合し、
3. 最終的に機能ごと（データ／セレクト作品／ブックシェルフ／アーカイブ一覧／
   ドロワー／リーディングルーム／言語切替 など）にきれいな関数群へ
   再編成する

という手順で進めるのが安全です。ご希望があれば、この続きの工程も
（1機能ずつ、変更前後の差分を明示しながら）進めます。

## 動かし方

`index.html` をブラウザで直接開くだけです（サーバー不要、元のファイルと
同じ仕組みです）。
